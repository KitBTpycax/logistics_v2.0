# Logistics Routing Application (EUTS)

Система для розрахунку та оптимізації логістичних маршрутів (Vehicle Routing Problem with Time Windows - VRPTW). Додаток розроблено на базі Flask і дозволяє автоматизувати процес побудови маршрутів доставки з урахуванням часових вікон, об'єму вантажу, кількості доступних рейсів та витрат на пальне/зарплату.

## Основні можливості

- **Оптимізація маршрутів (Multi-Trip VRPTW)**: Розподіл замовлень між водіями з мінімізацією загальних витрат (пальне + зарплата), використовуючи Google OR-Tools.
- **Управління користувачами**: Рольова система доступу.
  - *Логіст*: Розрахунок маршрутів, перегляд історії, призначення водіїв на маршрути, експорт в Excel.
  - *Водій*: Перегляд призначених йому маршрутів (Driver Dashboard).
- **Врахування витрат**: Гнучкі налаштування вартості пального (бензин, дизель, газ), зарплати водія, максимальної тривалості зміни та кількості рейсів.
- **Експорт**: Можливість завантажити детальний звіт по маршрутах у форматі Excel.
- **Гнучка база даних**: Підтримка SQLite для локальної розробки та PostgreSQL для production-середовища.

## Технологічний стек

- **Backend**: Python 3, Flask
- **База даних**: SQLAlchemy (PostgreSQL / SQLite)
- **Автентифікація**: Flask-Login, Flask-Bcrypt
- **Оптимізація (VRP)**: Google OR-Tools
- **Обробка даних**: Pandas, Openpyxl
- **Frontend**: HTML, CSS, Jinja2 (Flask Templates)

## Початок роботи

### Передумови

- Python 3.9+
- Встановлений PostgreSQL (якщо планується використання, а не SQLite)

### Встановлення та налаштування

1. **Клонуйте репозиторій:**
   ```bash
   git clone <URL_репозиторію>
   cd EUTS
   ```

2. **Створіть та активуйте віртуальне середовище:**
   ```bash
   python -m venv .venv
   # Для Windows:
   .venv\Scripts\activate
   # Для Linux/Mac:
   source .venv/bin/activate
   ```

3. **Встановіть залежності:**
   ```bash
   pip install -r requirements.txt
   ```

4. **Налаштуйте змінні середовища:**
   Створіть файл `.env` у кореневій папці проекту (або використовуйте існуючий) і додайте необхідні змінні. Приклад:
   ```env
   SECRET_KEY=your_secret_key_here
   DATABASE_URL=postgresql://user:password@localhost:5432/logistic_db
   # Якщо DATABASE_URL не вказано, система автоматично створить локальну SQLite БД (logistic.db)
   ```

5. **Ініціалізація бази даних:**
   База даних та початкові користувачі (1 логіст і 3 водії) ініціалізуються автоматично при першому запуску додатку (через `init_db` в `models.py`).

6. **Запуск додатку:**
   ```bash
   python app.py
   ```
   Додаток буде доступний за адресою: `http://127.0.0.1:5000`

## Користувачі за замовчуванням (Тестові дані)

При першому запуску в базу даних автоматично додаються тестові користувачі:

- **Логіст**:
  - Логін: `logist`
  - Пароль: `1234`
- **Водії**:
  - Логін: `driver1` (також `driver2`, `driver3`)
  - Пароль: `1234`

## Структура проекту

- `app.py` — Головний файл додатку Flask (роути, конфігурація).
- `models.py` — Моделі бази даних SQLAlchemy (User, Calculation, RouteAssignment).
- `services.py` — Бізнес-логіка (VRPSolver, DataProcessor, ReportService, DistanceService).
- `config.py` — Налаштування додатку та підключення до БД.
- `templates/` — HTML-шаблони.
- `static/` — Статичні файли (CSS, JS, зображення).
- `requirements.txt` — Список Python-залежностей.
